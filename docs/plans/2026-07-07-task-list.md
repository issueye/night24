# Task List Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a task list workflow so complex AI tasks are decomposed into visible steps, executed step by step, and closed with a completion report.

**Architecture:** The agent-core system prompt defines a stable Markdown contract for complex tasks. The React client parses assistant messages for that contract and renders a compact task progress panel above the conversation. The implementation stays local to prompt generation, message-derived UI state, and focused tests.

**Tech Stack:** Rust agent-core, React 18, Vite, Node test runner, lucide-react icons, existing CSS variables.

---

## Requirements

- For complex tasks, the model must first produce a `## 任务列表` section with Markdown task checkboxes.
- The model must work through the listed tasks and update task status as work progresses.
- The model must finish with a `## 完成报告` section that summarizes completed work, verification, and remaining notes.
- The UI must make the latest task list and completion report easy to scan without requiring the user to hunt through long assistant messages.
- The feature must preserve existing chat history, streaming deltas, tool call rendering, and session replay behavior.

## Design

The backend prompt will describe when to use the workflow and the exact output shape:

```markdown
## 任务列表
- [ ] Step 1
- [ ] Step 2

...work updates...

## 任务列表
- [x] Step 1
- [ ] Step 2

## 完成报告
- 已完成: ...
- 验证: ...
- 备注: ...
```

The frontend will derive task state from messages instead of storing a new mutable task model. This avoids database migrations and keeps recovery simple: when a session is reopened, parsing the saved assistant messages recreates the task list and report.

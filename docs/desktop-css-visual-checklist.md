# Night24 桌面端 CSS 迁移视觉检查清单

日期：2026-07-05

## 目的

为后续将 `tauri-app/src/styles/01-styles.css` 到 `06-styles.css` 迁移为功能化 CSS 文件建立最小视觉基线。每次迁移一个样式分组后，至少按本清单检查一次，避免编号文件拆分导致布局、弹窗、右侧面板或消息流回归。

## 检查前准备

1. 在 `tauri-app` 目录运行 `npm run build`，确认生产构建通过。
2. 使用 `npm run dev` 或 `npm run preview` 打开桌面端页面。
3. 至少检查两个视口：
   - 桌面宽屏：`1440 x 900`
   - 窄屏窗口：`1024 x 768`
4. 记录当前迁移范围，例如 `settings.css`、`chat.css` 或 `subagents.css`。

## 全局布局

- 顶部栏、左侧菜单、聊天主区域、右侧上下文区域之间没有重叠。
- 左侧菜单底部设置入口仍贴近菜单底部，不被会话列表挤出。
- 当前工作区、server/core 状态、重启 core 按钮在顶部栏内对齐稳定。
- 窗口缩窄后，主要按钮文字不溢出，不遮挡相邻图标。
- 主题、字体大小、访问模式等全局设置切换后，页面没有明显闪烁或布局跳动。

## 聊天与任务流

- 空会话状态、用户消息、助手消息、错误消息各自样式清晰可区分。
- Markdown 内容、代码块、列表、链接在消息气泡内不横向溢出。
- 输入框、多行输入、发送按钮、停止按钮在运行中和空闲态尺寸稳定。
- 任务运行中出现的 loading、事件状态、上下文阈值提示不会遮挡输入区。
- 历史消息滚动到底部时，最后一条消息和输入区之间保留可读间距。

## 权限与工具事件

- 权限确认卡片在聊天流中完整显示工具名、风险、摘要和操作按钮。
- Approve / Deny 按钮在窄屏下不换行到不可读状态。
- 工具调用块的展开、收起、错误态、成功态颜色和间距稳定。
- 事件时间线打开后不遮挡设置弹窗或输入区。

## 右侧上下文面板

- 文件树、文件预览、Diff、事件、子代理 tab 切换时内容区域高度稳定。
- 文件树长路径有截断或换行策略，不撑破右侧面板。
- Diff 面板 loading、空状态、错误态和变更列表都保持可读。
- Context token 阈值状态在标红位置显示清晰，不和相邻状态文字重叠。

## 设置弹窗

- 设置弹窗使用小圆角，弹窗和页面背景层级明确。
- 基础设置、Provider、Hook、Skill 切换时左侧列表和右侧详情高度一致。
- Provider 配置中的输入框、保存按钮、profile 列表不溢出弹窗。
- Hook 设置能显示 event、engine、entrypoint、timeout、instruction limit、allowed modules。
- Skill 设置能显示技能列表、详情、调用名复制按钮和空状态。
- 弹窗关闭按钮和底部操作按钮在窄屏下仍可点击。

## 子代理面板

- 子代理统计区 total/running/completed/failed/cancelled 数字对齐稳定。
- 子代理列表项状态、名称、任务摘要、时间信息在窄屏下不重叠。
- 子代理详情中的消息、结果、错误和状态更新能完整显示。
- 空池状态和加载失败状态不破坏右侧面板布局。

## CSS 迁移顺序建议

1. `base.css`：全局变量、reset、基础按钮和输入。
2. `layout.css`：顶部栏、左侧栏、主布局、右侧面板外框。
3. `chat.css`：消息、输入区、Markdown、工具调用块、权限卡。
4. `settings.css`：设置弹窗、设置列表详情壳、Provider/Hook/Skill 表单。
5. `context.css`：文件树、预览、Diff、事件时间线、上下文状态。
6. `subagents.css`：子代理统计、列表、详情。

## 每次迁移的通过条件

- `npm run build` 通过。
- 本清单中本次影响范围的检查项无明显回归。
- `tauri-app/src/styles.css` 的导入顺序与 cascade 依赖一致。
- 迁移只移动或重命名相关样式，不混入无关视觉重设计。
